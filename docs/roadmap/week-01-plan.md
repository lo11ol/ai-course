## 核心规则（避免半途而废）
1) 每天只做“最小交付物”（MVD：Minimum Viable Deliverable）
2) 先交付可用版本，再优化表达
3) 每篇内容都要能复用：模板/清单/Prompt 至少占 50%

## 每天的固定写作流程（60-90 分钟版本）
1) 先写“你要给读者的结果”  
   - 例如：一个模板、一张表、5 条 Prompt
2) 再写“照做步骤”（编号清单）
3) 最后补“解释”和“常见坑”（最少 3 条）

---

## 每篇文章的统一骨架（复制即可）
你可以把下面骨架复制到每一篇新文章里：

1) 你会得到什么  
2) 前置条件（小白能做到的）  
3) 标准流程（步骤清单）  
4) 模板/示例（可复制）  
5) Prompt（≥5，可直接用）  
6) 常见坑与排查（≥3）  
7) 练习题（≥3）  
8) 10 分钟验收清单

## Daily Check（每天 3 分钟复盘）
当天完成后，把对应 Day 打勾，并回答 3 个问题（写一句话即可）：
- [ ] D1 完成
- [ ] D2 完成
- [ ] D3 完成
- [ ] D4 完成
- [ ] D5 完成
- [ ] D6 完成
- [ ] D7 完成

复盘三问：
1) 我今天交付了什么“可复用”的东西？（模板/清单/Prompt）
2) 哪一步最耗时？下次如何更快？（例如先写模板再写解释）
3) 明天我只做一件事的话，应该做什么？
