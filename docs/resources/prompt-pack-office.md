## 使用说明（30 秒上手）
你只要做两件事：
1) 复制某条 Prompt  
2) 把里面的【变量】替换成你的真实内容（或把材料粘进去）

建议你把常用的 3-5 条收藏起来：写周报、写邮件、会议纪要、PPT 大纲。

---

## 万能提问框架（最重要的一条）
当你不知道怎么问时，先用这个框架。

### Framework 0：R-G-M-F-B（角色 / 目标 / 材料 / 格式 / 边界）
你是【角色】。  
我的目标是【目标】。  
以下是材料（请完整阅读再回答）：  
【材料粘贴区】

请按以下格式输出：  
【输出格式要求：表格/清单/JSON/逐页PPT等】

限制条件：  
- 受众是【受众：老板/客户/同事/学生】  
- 语气是【语气：专业/友好/强硬/简洁】  
- 篇幅是【篇幅：200字/一页/10条】  
- 禁止【禁区：夸大/编造数据/泄露隐私】  
如果材料不足，请先用问题列表向我确认。

## A. 会议（Meeting）
### Prompt 1：结构化会议纪要（通用版）
你是我的会议助理。请根据会议材料输出“可执行”的会议纪要。  
要求：
1) 先输出【结论摘要】3-7条（结论/决定，而不是复述）
2) 输出【行动项表格】，列：事项（动词开头）/负责人/截止时间/验收标准/优先级(高/中/低)
3) 输出【风险与依赖】（缺信息就列“需要补充的信息”并提问）
4) 输出【待确认问题】（Open Questions）
5) 全文中文，简洁，不说官话

会议材料如下：
【会议名称】
【时间】
【参会人员】
【目标】
【转写/聊天记录/笔记原文】

---

### Prompt 2：行动项质量检查（防止“写了等于没写”）
请检查下面的行动项是否可执行，并重写成更清晰的版本。  
标准：
- 动词开头
- 有负责人、截止时间、验收标准
- 不允许“尽快/看情况/跟进一下”这类模糊词
输出：
1) 问题清单
2) 修改后的行动项表格（同字段）

行动项如下：
【粘贴行动项】

## B. 邮件与消息（Email & Messaging）
### Prompt 4：写一封“清晰且不尴尬”的邮件
你是我的职场写作助手。请帮我写一封邮件。  
输入变量：
- 收件人类型：【老板/客户/同事/老师】
- 目的：【汇报/请求支持/催进度/道歉/通知】
- 背景：【3-5句背景】
- 我希望对方做什么：【明确动作】
- 截止时间：【日期 时间】
- 附件/链接：【如有】

输出格式：
- 标题（Subject）
- 正文（不超过 180 字，分段清晰）
- 结尾一句“下一步/期待确认”

注意：
- 语气【礼貌且坚定/非常客气/强硬一点】
- 不要夸大，不要编造数据

---

### Prompt 5：催进度（不冒犯版）
请帮我写一段“催进度”消息，适合发在群里或私聊。  
变量：
- 对方是谁：【姓名/角色】
- 我催的事项：【事项】
- 原承诺时间：【时间】
- 当前影响：【影响一句话】
- 我希望对方回复什么：【给出新时间/给出阻塞原因/给出进展】

输出 2 个版本：
1) 友好版（适合关系一般）
2) 坚定版（适合项目紧急）

---

### Prompt 6：把一段话改写成“更专业、更清楚”
请把下面这段话改写成更专业、更清楚的版本。

- 逻辑顺序：背景 → 结论 → 下一步
- 给出 3 个不同语气版本：正式/中性/更强势

原文：
【粘贴原文】

## 推荐用法：一条龙工作流（最省脑）
1) 先用 Framework 0 把“目标/材料/格式/边界”说清楚
